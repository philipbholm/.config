# Learnings from PR #271
Removed an unused "variable type" concept from the codebase, deleting dead code across 21 files to simplify the data model until the feature is actually needed.

---

## 1. Rebase and Fix Tests Before Merging After Concurrent Changes
When working on a removal or refactoring PR that touches many files, other teammates may merge changes to the same codebase in parallel. If their merge introduces or modifies tests that reference the code you are removing, your PR will have broken tests even if it was green when you opened it. Merging without re-running tests risks introducing failures into the main branch, which blocks the entire team.

**Guideline:** Always rebase onto the latest main branch and verify all tests pass immediately before merging, especially on large-scope removal PRs.

---

## 2. Remove Dead Code Promptly Rather Than Leaving It for Later
This PR removed a concept ("variable type") that was defined across many files but never actually used. Unused abstractions increase cognitive load for every developer who reads the code, create false dependencies that complicate refactors, and risk being half-implemented by someone who mistakes them for active features. The longer dead code stays, the harder it is to confidently remove.

**Guideline:** Delete unused types, interfaces, and abstractions as soon as they are confirmed unnecessary rather than leaving them as placeholders for speculative future work.

---

## 3. Scope Removal PRs to a Single Concept for Clean Review
This PR cleanly targeted one concept (variable type) for removal across 21 files. Mixing a removal with new feature work or unrelated refactors would make it much harder for reviewers to verify that nothing important was lost. A focused removal PR makes the diff self-explanatory: every deletion relates to the same unused concept, and reviewers can approve with confidence.

**Guideline:** When removing a cross-cutting concept, create a dedicated PR that does nothing but the removal so reviewers can verify completeness without distraction.

# Learnings from PR #1206

This PR refactored e2e tests for the studies feature by introducing a Page Object Model pattern and a `StudyBuilder` utility, replacing scattered test setup code with reusable page classes and a fluent builder API.

---

## 1. Extract Shared Configuration as Constants Rather Than Requiring Injection

When a value like a base API URL is the same in every test, make it a default export or constant rather than requiring every call site to inject it. Unnecessary injection adds boilerplate without providing real flexibility.

```typescript
// Instead of requiring callers to pass the URL every time:
const client = createTestClient("http://localhost:4000");

// Export a constant and use it as the default:
import { createTestClient, E2E_API_URL } from "test-util/e2e-gql";
const client = createTestClient(E2E_API_URL);
```

The feedback was: "why we need to inject this at all -- maybe make it default or something." If the URL never varies, a default parameter or module-level constant eliminates redundancy and reduces the chance of typos.

**Guideline:** When a configuration value is constant across all usages, export it as a named constant and use it as the default parameter rather than requiring explicit injection at every call site.

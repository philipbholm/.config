# Learnings from PR #1712

This PR added codelist cleanup when deleting a registry that has codelist enabled, including graceful NOT_FOUND handling and integration tests.

---

## 1. Order Cross-Service Side Effects After the Primary Event Publish

When a use case performs a side effect on an external service (e.g., deleting codelist data) *and* publishes a domain event (e.g., `REGISTRY_DELETED`), the ordering matters for failure recovery. If the side effect runs first and event publication fails, the system is left in an inconsistent state: the external resource is gone but the primary entity still exists. Reversing the order (publish event first, then perform cleanup) means a failure during cleanup leaves orphaned data in the external service -- which is less harmful because the primary operation completed and orphaned data can be cleaned up later.

The proper solution for coordinating multi-service operations is a saga or transactional outbox pattern, but when that is not available, prefer the ordering that leaves the system in the least harmful inconsistent state on failure.

**Guideline:** When a use case both publishes a domain event and calls an external service for cleanup, publish the event first so that a cleanup failure leaves orphaned data rather than a primary entity missing its dependent resources.

---

## 2. Keep Test Stub Defaults Consistent Across All Methods

When a test helper provides a stubbed service where most methods reject with "Not implemented" by default, newly added methods should follow the same convention. If a new method silently resolves by default while others reject, tests that accidentally trigger that method will pass without revealing unintended side effects. Override the default only in the specific tests that require the method to succeed, using an explicit spy or mock.

**Guideline:** Default all stubbed service methods to reject with "Not implemented" and override to resolve only in tests that explicitly exercise that method, so accidental invocations surface as test failures.

## 1. Register New API Scopes in Infrastructure When Adding Endpoints

When a new API operation is introduced (e.g., `codelist/record.get_history`), the corresponding authorization scope must be registered in the infrastructure layer. Forgetting this step means the endpoint may be unreachable or improperly gated in deployed environments, even if it works locally with stubbed auth.

**Guideline:** Whenever you add a new backend operation, check whether a corresponding scope or permission needs to be registered in the infrastructure/auth configuration and include that change in the same PR.

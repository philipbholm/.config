# Learnings from PR #843

This PR relaxed the last name validation regex to allow spaces (e.g., "Doe Smith") in addition to letters and hyphens.

---

## 1. Trim and Normalize Input When Relaxing Validation Constraints

When broadening a regex or validation rule to accept new characters (like spaces), consider whether the input should be trimmed or normalized first. Allowing `\s` in a regex without trimming means inputs like `"  John  "` or `"John D     oe"` (multiple consecutive spaces) will pass validation but produce data quality issues. Apply `.trim()` before validation, and consider collapsing consecutive whitespace.

**Guideline:** Whenever you relax a string validation to accept whitespace, add a `.trim()` step (or Zod `.transform(s => s.trim())`) before the regex check to prevent leading/trailing/excessive whitespace from slipping through.

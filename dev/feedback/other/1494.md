# Learnings from PR #1494

Removed multi-site registry functionality from the application, consolidating all registry operations into a single registry type (601 files changed, -13,000 lines).

---

## 1. Update Error Messages and Strings When Renaming Routes or Concepts

After removing the "single-site-registry" path segment from routes, an error message in `useFormId.ts` still referenced the old route path `single-site-registry/[registryId]/registry-design/forms/[formId]`. When renaming or removing concepts that appear in string literals (error messages, logs, comments), search the codebase for all textual references -- not just imports and code paths.

**Guideline:** After a large rename or removal, grep for the old name in string literals, comments, and error messages to catch stale references that the compiler will not flag.

---

## 2. Disambiguate When Multiple Implementations of the Same Hook Exist

A reviewer asked "Which one?" about a `useRegistryId` import, because the codebase had multiple implementations of hooks with the same name under different route directories. After consolidation, the correct import was confirmed, but the ambiguity itself signals that having multiple implementations of identically-named hooks creates confusion.

**Guideline:** After consolidating duplicate code paths, verify that only one canonical implementation of each shared hook or utility remains, and remove or redirect the duplicates.

---

## 3. Keep Large Mechanical Refactors Reviewable

This PR changed 601 files. A reviewer resorted to asking an AI bot to summarize the changes because the diff was too large to review manually. While mechanical deletions may be low-risk individually, the sheer volume makes it easy for subtle regressions (like stale route references) to slip through.

**Guideline:** For large-scale mechanical refactors, consider breaking the PR into logical chunks (e.g., delete unused code in one PR, update remaining references in another) or provide a structured summary of non-obvious changes to aid review.

# Learnings from PR #1046

Updated to React 19, added new UI components (Input, PasswordInput, ButtonGroup), fixed an i18n bug with Zod, and fixed a dark mode UI issue.

---

## 1. Avoid Excessive Default Styling in Reusable Components

Baking default layout styles (flex, gaps, margins) directly into reusable components makes them convenient initially but creates friction when consumers need different layouts. You end up fighting the defaults with overrides. Prefer keeping components minimally styled and letting the consuming code control layout.

**Guideline:** Keep reusable components free of layout opinions (flex direction, gaps, margins); let the parent control positioning and spacing.

---

## 2. Deprecate Old Components When Introducing Replacements

Adding a new `Input` component alongside an existing one without deprecating the old version creates ambiguity about which to use. New code will split between both, and the old one will never get cleaned up. Mark the old component as deprecated immediately so consumers know to migrate.

**Guideline:** When adding a replacement component, mark the old one as `@deprecated` with a pointer to the new one in the same PR.

---

## 3. Avoid Unnecessary React.memo

Wrapping components in `React.memo` adds cognitive overhead and code clutter. Unless profiling shows a specific re-render performance problem, the memoization is premature optimization. In most cases, React's default rendering is fast enough, and memo just makes the code harder to read.

**Guideline:** Only use `React.memo` when profiling identifies a measurable re-render performance issue, not as a default practice.

---

## 4. Avoid Hardcoded Default IDs on Components

Giving a component a hardcoded default `id` prop means multiple instances on the same page will share the same ID, breaking HTML validity and causing issues with label associations and test selectors. IDs should be explicitly passed by the consumer or omitted entirely.

**Guideline:** Never set a default `id` on a reusable component; require callers to provide one explicitly if needed.

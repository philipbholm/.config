# Learnings from PR #827

This PR added filter matching support (exact case-insensitive matching) when searching codelist records.

---

## 1. Keep Domain Layer Independent of Transport Layer Types

When defining types in the domain/application layer (e.g., `FilterMatch` in `search-utils.ts`), do not import from the gRPC/transport layer (e.g., generated proto enums). The domain layer should own its own type definitions, and the handler/transport layer is responsible for mapping between its types and the domain types. Importing proto types into the domain creates a coupling that violates layered architecture.

**Guideline:** Domain layer types must be self-contained; map between transport (gRPC, REST) types and domain types at the handler boundary, never import transport-generated types into application code.

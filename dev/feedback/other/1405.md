# Learnings from PR #1405

Introduced AGENTS.md and local rule files (.cursorrules) to standardize coding conventions for AI-assisted development across the monorepo.

---

## 1. Prefer IDE-Agnostic Formats for AI Agent Instructions

When adding AI coding conventions to a shared repository, tool-specific formats (like `.cursorrules` for Cursor) lock out team members using other editors. A reviewer suggested using "Agent Skills" or a generic format so that instructions work regardless of IDE. The PR ended up creating both `AGENTS.md` and `CLAUDE.md` (as a symlink), which is a step in the right direction, but the `.cursorrules` files remained Cursor-specific.

**Guideline:** Store AI agent instructions in tool-agnostic markdown files (e.g., `AGENTS.md`) and symlink or convert for specific tools, rather than committing tool-specific rule files as the source of truth.

---

## 2. Avoid Duplicating Rules Across Multiple Locations

The same naming convention appeared in three different files with slightly different wording (general rules, frontend rules, and registries-specific rules). When a rule applies broadly, duplicating it in domain-specific files creates drift and confusion about which version is authoritative.

**Guideline:** Define each convention in exactly one place and reference it from domain-specific files rather than restating it with different wording.

---

## 3. Co-locate Dictionaries Unless Sharing Is Required

A reviewer asked whether the AI rules should instruct keeping dictionaries in the same file as the component rather than in separate `dictionary.ts` files. The response was that sharing is sometimes needed, so a blanket rule would be wrong. This reveals a nuanced pattern: co-location is the default, but shared dictionaries are a valid exception.

**Guideline:** Default to co-locating dictionaries with their consuming component, and only extract to a shared file when multiple components genuinely need the same translations.

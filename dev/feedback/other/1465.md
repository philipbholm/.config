# Learnings from PR #1465

This PR bumped the `qs` package to a safe version to address multiple HIGH-severity vulnerability issues across several services.

---

## 1. Prefer `npm audit fix` Over Manual Overrides for Vulnerability Fixes

When fixing vulnerable transitive dependencies, prefer using `npm audit fix` to let npm resolve the dependency tree naturally, rather than manually adding `overrides` in package.json. Manual overrides can produce different lock file results than the standard tooling and may confuse future contributors.

**Guideline:** Use `npm audit fix` as the first approach for vulnerability remediation; only resort to manual overrides when audit fix cannot resolve the issue.

---

## 2. Document Override Entries with CVE References and Removal Conditions

If you must use a manual `overrides` entry in package.json, add a comment (using the `//key` convention) explaining which CVE it addresses and when it can be safely removed. Without this context, overrides accumulate as mystery configuration that nobody dares to touch.

```json
"overrides": {
  "//qs": "CVE-2022-24999 fix, remove when parent deps update",
  "qs": ">=6.14.1"
}
```

**Guideline:** Every manual dependency override must include a comment with the CVE reference and a condition for when the override can be removed.

# Learnings from PR #1027

Added visits concept to group related forms in the registry system, with visit CRUD and form-to-visit assignment.

---

## 1. Model All Access Through Explicit Permissions

Even if a permission will not be exposed in the UI initially, every access path should be backed by an explicit permission model. This avoids implicit access that becomes hard to audit or restrict later. Whether or not the UI surfaces it is a separate decision from whether the backend enforces it.

**Guideline:** Create an explicit permission entry for every resource access path, even if the UI does not yet expose permission management for it.

---

## 2. Specify Expected Error Messages in Rejection Assertions

Using `expect(...).rejects.toThrow()` without specifying the error message means any thrown error will make the test pass, including unrelated failures. This creates false positives where the test passes for the wrong reason.

**Guideline:** Always pass the expected error message or class to `.toThrow("specific message")` to ensure the test validates the correct failure mode.

---

## 3. Do Not Commit TODO Comments

Leaving TODO comments in committed code creates invisible tech debt that has no tracking or accountability. If something needs to be done, either do it in the PR or create a ticket and reference it. Committed TODOs tend to linger indefinitely because they are not surfaced in any project management tool.

**Guideline:** Never commit code with TODO comments; either fix the issue in the same PR or create a tracked ticket and remove the comment.

---

## 4. Use Translation Keys for All User-Facing Strings

Hardcoded strings in UI components break internationalization and make it impossible to support multiple languages without code changes. Every user-visible string should go through the translation system from the start, because retrofitting translations later is significantly more expensive.

**Guideline:** Always use translation keys (e.g., `t("visits.title")`) for any text rendered in the UI, never hardcoded strings.

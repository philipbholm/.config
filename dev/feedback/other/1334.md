## 1. Use `size-{n}` Instead of Separate `h-{n} w-{n}` in Tailwind

When an element should be square (equal width and height), use Tailwind's `size-{n}` utility instead of specifying `h-{n}` and `w-{n}` separately. It is more concise, easier to scan, and reduces the chance of accidentally changing only one dimension.

```tsx
// Prefer this:
<Trash2 className="size-4" />

// Over this:
<Trash2 className="h-4 w-4" />
```

**Guideline:** Use `size-{n}` for square elements to keep Tailwind classes concise and to avoid dimension mismatches.

# Learnings from PR #1106

This PR formatted ICD-10 codes for correct display (e.g., A00.1 instead of A001) and added a field for tracking unspecified diagnosis codes.

---

## 1. Disconnect Database Clients in Standalone Scripts

A seed script (`seed-db.ts`) that creates a Prisma client and writes to the database did not call `prisma.$disconnect()` at the end. Without disconnecting, the Node.js process holds open database connections and hangs unnecessarily after the work is done, requiring a manual kill.

```typescript
// At the end of the script:
await prisma.icd10Code.createMany({ data });
await prisma.$disconnect(); // prevents process from hanging
```

**Guideline:** Always call `prisma.$disconnect()` (or equivalent cleanup) at the end of standalone scripts and seed files so the process exits cleanly.

---

## 2. Add Tests That Assert New Fields Actually Work, Not Just Exist

When adding a new field (`unspecified: false`) to test fixtures, a reviewer noted that simply including the field in seed data without any test cases that assert behavior around it is insufficient. If the field matters, there should be tests that verify filtering, querying, or display logic that depends on the new field.

**Guideline:** When adding a new data field, include at least one test that exercises the logic dependent on that field (e.g., filtering unspecified codes, aggregating them), not just test fixtures that happen to include it.

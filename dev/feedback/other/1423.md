# Learnings from PR #1423

Adding a new registry template (Nobarev) with test data generation and template infrastructure changes.

---

## 1. Clean Up Stale Naming When Refactoring Shared Code

When modifying shared infrastructure like template registries or indexes, take the opportunity to remove outdated naming conventions and suffixes. Leaving stale names (like a `-registry` suffix that no longer applies) creates inconsistency and confuses future contributors about what the canonical naming pattern is.

**Guideline:** When touching shared code, audit neighboring names for consistency and remove legacy naming artifacts in the same PR.

---

## 2. Add Unit Tests for Data Generation Logic

Test data generators and seed scripts contain logic that can silently produce invalid data if broken. Even though they are used in development/testing contexts rather than production, incorrect test data leads to false confidence in integration tests and hard-to-trace bugs.

**Guideline:** Write unit tests for any data generation or transformation logic, especially when it encodes domain rules like valid value ranges or entity relationships.
